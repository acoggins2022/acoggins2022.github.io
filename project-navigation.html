Project Case Study: Indoor Navigation System

A cloud-based solution to enhance independence for individuals with mobility impairments.

The Problem

"Navigating unfamiliar indoor spaces poses significant challenges for individuals with mobility disabilities, often resulting in dependency and reduced quality of life. This project proposes an innovative solution designed to enhance the independence and safety of this demographic through a sophisticated navigational aid system."

My Role & Contributions

As the lead backend developer, I was responsible for designing and implementing the cloud infrastructure on AWS. My key role was architecting the cloud solution with a focus on security, including a PostgreSQL database in a secure VPC using AWS RDS and a serverless API with Python/Flask, ensuring least-privilege access with IAM roles. I also co-authored the literature review and wrote the final decision justification for our technology stack.

Database Design

I designed the PostgreSQL database schema from the ground up to efficiently store spatial data, user information, and navigation routes. The design ensures data integrity and supports complex queries needed for pathfinding.

Technical Challenges & Lessons Learned

A key lesson was in pathfinding optimization. The A* algorithm performs well in static settings, but we identified that it struggles with real-time recalculations when encountering moving obstacles. Future iterations would benefit from exploring probabilistic path planning methods like D* or RRT to enhance adaptability in dynamic environments.
